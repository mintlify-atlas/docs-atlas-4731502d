## What is Low-Poly Ocean?

Low-Poly Ocean is an immersive, scroll-driven web experience that uses **polygon (low-poly) art** to tell the story of 10 endangered marine animals found in Thai waters. Each animal is rendered as a mosaic of triangular shards that assemble before your eyes — a visual metaphor for how ecosystems shatter when a species disappears.

The project is built with **React 19**, **Framer Motion**, and **Three.js**, and is deployed as a static site via Vercel. Content is written in Thai and covers three topics for each animal: an introduction to the species, the threats it faces, and conservation solutions.

## Key Features

<AccordionGroup>
  <Accordion title="Low-poly polygon art system">
    Each animal is defined as an array of triangular shards in `src/data/animalConfig.js`. Every shard has a `clipPath` (polygon coordinates as percentages), a `color`, and an `order` value that controls assembly sequence. The system supports hundreds of shards per animal for high-fidelity artwork.
  </Accordion>
  <Accordion title="Animated shard assembly">
    On the Detail page, shards fly in from scattered positions and snap into place with staggered CSS transitions. The total assembly time is normalized to ~1.25 seconds regardless of how many shards an animal has.
  </Accordion>
  <Accordion title="Parallax scrolling ocean">
    The Ocean Explorer page uses Framer Motion's `useScroll` and `useTransform` hooks to drive per-element parallax. Animals enter from off-screen, drift through the viewport, and fade out — creating a sense of depth and motion.
  </Accordion>
  <Accordion title="Three.js underwater environment">
    A `UnderwaterEnvironment` component renders a real-time 3D background using `@react-three/fiber` and `@react-three/drei`, complete with caustic light effects.
  </Accordion>
  <Accordion title="Ambient ocean soundscape">
    A `SoundController` component manages background audio with user-controlled play/pause toggle, respecting browser autoplay policies.
  </Accordion>
  <Accordion title="Responsive design">
    All pages adapt to mobile viewports. The Stages hub uses a tap-to-preview interaction on small screens. The depth indicator and scroll scrubber are optimized for touch.
  </Accordion>
</AccordionGroup>
